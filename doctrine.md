# Doctrine — what the founders enforce

These are culture rules for **humans**, not agents (agent permissions live in `rules.md`). The technology in this repo fails inside a default company. These ten choices are what make it work. They are uncomfortable on purpose.

1. **Record everything by default.** Meetings, decisions, user conversations. "If it is recorded, it happened to the AI. If it did not get recorded, it did not happen to your intelligence."
2. **One context store.** This repo is the company's memory. No knowledge in private silos, no decisions that live only in someone's head or a WhatsApp thread.
3. **Open by default.** Agent sessions and their logs are visible to the whole team. People learn from watching; transparency replaces lockdown as the security model.
4. **Give agents real access.** Read-only on real data first, then earn write access per workflow. Safetyism is the tax that keeps companies two years behind.
5. **Burn tokens, not headcount.** Budget real money for agent usage. It buys you the way work will feel in two years, today.
6. **Skillify as a ritual.** Anything done twice becomes a skill file. Skills improve from the transcripts of using them.
7. **Flatten.** Individual contributors + one named owner (DRI) per workflow. No committees.
8. **Context is the asset; software is disposable.** Protect and version the brain. Regenerate scripts and dashboards whenever models improve.
9. **Everyone owns their prompts.** Anyone on the team can read and edit the skills and rules behind the tools they use.
10. **Run the nightly review.** If no one (human or agent) reviewed yesterday's failures, the company didn't learn yesterday.

The litmus test, asked every quarter: **"If you were building this company today, would you start it in this shape?"**
