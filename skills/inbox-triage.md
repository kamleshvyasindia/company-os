# Skill: inbox-triage

**Trigger:** nightly. **Goal:** inbox items become brain pages; the inbox stays a clean chronological record.

1. List `inbox/` files not yet marked in `logs/triage-log.md`.
2. For each: classify (meeting / customer signal / idea / decision / reference) → create or update the right brain page(s), linking back to the inbox source. Search before creating — one page per entity, no duplicates.
3. Anything unclassifiable: leave it, note it in the triage log with a question for the morning brief.
4. Append processed files + actions taken to `logs/triage-log.md`.

**Never:** edit or delete inbox files. **Gate:** when a triaged item contradicts an existing brain page, do NOT silently overwrite — flag the contradiction in the morning brief.
